# 挑战定义

让**想转行做产品经理但零基础的人**，在**刷科普视频时**，搞清楚**产品开发的完整工作流是什么、PM 到底在做什么**。

## 核心要点
1. PM 不是写需求文档，而是「翻译官」——把用户痛点翻译成产品方案
2. 产品开发 5 步走：发现问题 → 定义需求 → 设计方案 → 开发上线 → 迭代优化
3. PM 的核心技能是「判断优先级」，而不是会画原型图

## 感性材料清单
- 案例：微信「拍一拍」功能的诞生——用户调研发现大家想要一种「无压力打招呼」的方式
- 类比：PM 像餐厅经理，不需要会炒菜（写代码），但要知道客人想吃什么、协调厨师和服务员
- 数据：一个功能从立项到上线平均需要 6-8 周，其中需求评审占 40% 时间
- 场景：你每天用的外卖 App，「预计送达时间」这个小功能背后，PM 要协调算法、设计、前后端四个团队

## HKRR 评估
- H（快乐）：用「餐厅经理」类比，轻松不烧脑
- K（知识）：产品开发 5 步工作流 + PM 真实职责
- R（共鸣）：戳中转行焦虑——「不会写代码能做 PM 吗？」
- R（节奏）：60 秒，8 个分镜，每镜 6-10 秒，节奏紧凑
