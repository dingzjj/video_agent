# 开场钩子设计指南

## 钩子的作用
前 3-8 秒决定用户是否继续看。钩子必须制造"现在就该看"的理由。

## 五种钩子模式

### 1. 痛点唤醒型
直接戳中用户正在经历的困扰。

**公式**：你是不是也 [痛点]？
**示例**：
- "你是不是也搞不清 RAG 和微调的区别？"
- "稳定币到底稳不稳？很多人都被这个问题困扰"

### 2. 反常识型
挑战用户的固有认知。

**公式**：你以为 [常识]，其实 [真相]
**示例**：
- "RAG 不是检索，是推理"
- "稳定币最大的风险不是脱锚"

### 3. 承诺结果型
明确告诉用户看完能得到什么。

**公式**：[时长] 让你 [具体结果]
**示例**：
- "一分钟让你彻底搞懂 RAG 原理"
- "60 秒带你看清稳定币的三种模式"

### 4. 场景代入型
把用户拉进一个具体场景。

**公式**：想象一下 [场景]
**示例**：
- "想象你在和 ChatGPT 聊天，它突然引用了你上传的文档"
- "假如你手里有 1 万美元，想换成稳定币"

### 5. 数据冲击型
用反差数据制造好奇。

**公式**：[惊人数据] + 为什么
**示例**：
- "90% 的人在用 RAG，但只有 10% 真正理解它"
- "USDT 市值 1000 亿美元，但储备金从未被完整审计"

## 选择原则
- 概念解释类 → 痛点唤醒 / 反常识
- 技术原理类 → 场景代入 / 承诺结果
- 对比分析类 → 数据冲击 / 反常识
