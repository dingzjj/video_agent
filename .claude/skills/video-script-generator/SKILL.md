# 视频脚本生成器

## 触发条件
- 用户说"生成视频脚本" / "写个视频脚本" / "帮我拆解成脚本"
- 用户给了概念关键词或 URL

## 核心原则
1. 先定义挑战（谁+场景+问题），再写脚本
2. 感性材料 ≥60%（案例/类比/数据/场景）
3. 开场必须有钩子（3-8秒抓住注意力）
4. 画面描述和文本内容清晰对应
5. 至少满足 HKRR 中的 2 项（快乐/知识/共鸣/节奏）
6. 将"工具性内容"转化为"体验性故事"

## 工作流程

### Step 0: 读取 references（按需）
- 开场钩子设计 → `references/hook-patterns.md`
- 得到品控标准 → `references/dedao-standards.md`
- HKRR 理论 → `references/hkrr-theory.md`
- 何同学叙事结构 → `references/hetongxue-narrative.md`
- 输出格式 → `references/output-format.md`

### Step 1: 确认输入模式
问用户：
```
你想做什么类型的视频？

1. 概念解释（给我一个关键词，如"RAG原理"）
2. 文章改编（给我 URL 或文章内容）

回复序号就行。
```

### Step 2: 定义挑战
**必须先定义挑战，再开始写脚本。**

问用户三个问题：
1. 目标观众是谁？（如：想用 AI 的产品经理 / 加密货币新手）
2. 他们在什么场景下会看这个视频？（如：选型时 / 投资前）
3. 看完要解决什么问题？（如：搞清楚 RAG 能干什么 / 判断稳定币是否适合自己）

生成挑战定义：
```
让 [目标观众] 在 [场景] 下，[解决的问题]
```

如果用户说"随便" / "不知道"，根据主题推断一个默认挑战，输出给用户确认。

### Step 3: 内容调研
**概念模式**：
- 用 WebSearch 搜索该概念的权威解释
- 找 3-5 个具体案例/数据/类比
- 识别常见误解（用于"不是而是法"）

**脚本模式**：
- 用 WebFetch 抓取 URL 内容
- 提取核心观点（3-5 个）
- 找支撑材料（案例/数据）

### Step 4: 设计开场钩子
根据内容类型选择钩子模式（见 `references/hook-patterns.md`）：
- 概念解释 → 痛点唤醒 / 反常识
- 技术原理 → 场景代入 / 承诺结果
- 对比分析 → 数据冲击 / 反常识

生成 2-3 个钩子备选，输出给用户选择。

### Step 5: HKRR 检查点
在生成脚本结构前，确认至少满足 2 项：
- [ ] H（快乐）：是否有轻松/幽默的点？
- [ ] K（知识）：核心知识点是什么？（必选）
- [ ] R（共鸣）：是否触动情感/痛点？
- [ ] R（节奏）：时长分配是否合理？

### Step 6: 生成脚本结构（何同学框架）
按 1 分钟视频规划：

**开场（5-8秒）- Hook**：
- 钩子（用户已选择的）
- 制造悬念或预告结果
- visual: 大字标题 + 副标题

**核心内容（40-45秒）- Journey**：
- 拆分成 3-5 个要点
- 每个要点 8-12 秒
- 体验而非罗列（用场景代入）
- 每个要点必须有感性材料
- visual: 关系图/流程图/对比/高亮

**结尾（5-8秒）- Payoff**：
- Take Away（一句话总结）
- 升华到人文意义
- visual: 大字 + 简单图形

### Step 7: 填充画面和文本
为每个分镜填充：
1. **visual**（画面描述）
   - type: 画面类型
   - content: 具体显示什么（可执行级别）
   - background: 背景描述
   - animation: 动画效果

2. **subtitle**（字幕）
   - 精简版，每行 ≤15 字
   - 只保留核心信息

3. **voiceover**（口播）
   - 完整版，口语化
   - 包含连接词、补充细节
   - 像在和朋友聊天

### Step 8: 品控检查（三重标准）
生成脚本后，自检：

**得到标准**：
- [ ] 挑战是否清晰？
- [ ] 感性材料是否 ≥60%？
- [ ] 观众看完能否行动？

**HKRR 标准**：
- [ ] 至少满足 2 项（H/K/R/R）？
- [ ] K（知识）是否清晰？

**何同学标准**：
- [ ] 开场是否制造悬念？
- [ ] 过程是否有体验感？
- [ ] 结尾是否有升华？

如果有不合格项，修正后再输出。

### Step 9: 输出 script.json
按 `references/output-format.md` 格式生成 JSON 文件。

保存到：`output/[主题名]/script.json`

同时生成 `output/[主题名]/challenge.md`：
```markdown
# 挑战定义

让 [目标观众] 在 [场景] 下，[解决的问题]

## 核心要点
1. [要点1]
2. [要点2]
3. [要点3]

## 感性材料清单
- 案例：[具体案例]
- 类比：[类比说明]
- 数据：[关键数据]

## HKRR 评估
- H（快乐）：[具体体现]
- K（知识）：[核心知识点]
- R（共鸣）：[情感触点]
- R（节奏）：[时长分配]
```

## 输出目录结构
```
output/[主题名]/
├── challenge.md      # 挑战定义 + HKRR 评估
├── script.json       # 脚本文件
└── research.md       # 调研素材（可选）
```

## 常见问题

**Q: 用户只给了一个关键词，没有明确挑战怎么办？**
A: 根据关键词推断默认挑战，输出给用户确认。例如"RAG原理" → "让想用 AI 的开发者，搞清楚 RAG 的工作原理和适用场景"

**Q: 调研时找不到足够的感性材料怎么办？**
A: 优先用类比和场景代入。例如讲 RAG 时，用"图书管理员"类比，用"你在用 ChatGPT 时"场景代入。

**Q: 画面描述太抽象，video-producer 无法实现怎么办？**
A: visual.content 必须具体到可执行。不要写"展示原理"，要写"三个方框从左到右，箭头连接，标注文字"。

**Q: 如何平衡 HKRR 四要素？**
A: K（知识）是必选项，至少再选 1 项。推荐组合：K+R（共鸣）适合概念解释，K+H（快乐）适合技术科普。

**Q: 何时需要升华到人文意义？**
A: 不是所有视频都需要。1 分钟视频优先把知识讲清楚，升华是加分项不是必选项。
